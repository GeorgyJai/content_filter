# Bot Service

Telegram Bot для взаимодействия с пользователями системы персонализированных дайджестов.

## Описание

Bot Service предоставляет пользовательский интерфейс через Telegram Bot API для:
- Онбординга новых пользователей
- Управления подписками на источники контента
- Настройки предпочтений и расписания
- Получения персонализированных дайджестов
- Сбора обратной связи

## Функциональность

### Основные команды

- `/start` - начало работы с ботом, онбординг
- `/settings` - изменение настроек
- `/add_source` - добавление нового источника
- `/import_folder` - импорт папки каналов Telegram
- `/bulk_add` - массовое добавление каналов
- `/my_sources` - список подключенных источников
- `/help` - справка по командам

### Процесс онбординга

1. **Приветствие** - описание возможностей системы
2. **Интересы** - пользователь описывает свои интересы
3. **Источники** - добавление Telegram-каналов
4. **Расписание** - настройка частоты доставки дайджестов

### Импорт папок каналов (Release 2)

Новая функция позволяет быстро импортировать все каналы из папки Telegram:

1. **Создание ссылки на папку**:
   - Откройте папку каналов в Telegram
   - Нажмите на название папки → "Поделиться папкой"
   - Скопируйте ссылку (формат: `https://t.me/addlist/...`)

2. **Импорт**:
   - Отправьте ссылку боту
   - Бот автоматически извлечет все каналы из папки
   - Каналы будут добавлены в ваши источники

3. **Массовое добавление**:
   - Используйте `/bulk_add` для добавления списка каналов
   - Отправьте несколько ссылок (по одной на строку)
   - Бот обработает все каналы одновременно

### Настройки

- **Интересы** - изменение тематических предпочтений
- **Расписание** - выбор частоты доставки (4, 8, 12, 24 часа)
- **Детализация** - уровень подробности резюме (подробно, средне, кратко)

## Технологии

- **aiogram 3.x** - асинхронная библиотека для Telegram Bot API
- **Telethon** - для парсинга папок каналов Telegram
- **gRPC** - взаимодействие с User Service
- **Python 3.11+** - основной язык разработки

## Установка и запуск

### Локальная разработка

1. Установите зависимости:
```bash
pip install -r requirements.txt
```

2. Создайте `.env` файл на основе `.env.example`:
```bash
cp .env.example .env
```

3. Заполните переменные окружения:
```env
BOT_TOKEN=your_telegram_bot_token
TELEGRAM_API_ID=your_api_id
TELEGRAM_API_HASH=your_api_hash
USER_SERVICE_HOST=localhost
USER_SERVICE_PORT=50051
```

**Получение Telegram API credentials** (для импорта папок):
- Перейдите на https://my.telegram.org/apps
- Войдите с вашим номером телефона
- Создайте новое приложение
- Скопируйте `api_id` и `api_hash`

4. Сгенерируйте proto файлы (если еще не сделано):
```bash
cd ../../shared/proto
python -m grpc_tools.protoc -I. --python_out=. --grpc_python_out=. user_service.proto
```

5. Запустите бота:
```bash
python main.py
```

### Docker

Запуск через Docker Compose (из корня проекта):
```bash
docker-compose up bot-service
```

## Переменные окружения

| Переменная | Описание | Значение по умолчанию |
|-----------|----------|----------------------|
| `BOT_TOKEN` | Токен Telegram бота | - (обязательно) |
| `TELEGRAM_API_ID` | Telegram API ID (для папок) | `0` (опционально) |
| `TELEGRAM_API_HASH` | Telegram API Hash (для папок) | `""` (опционально) |
| `USER_SERVICE_HOST` | Хост User Service | `user-service` |
| `USER_SERVICE_PORT` | Порт User Service | `50051` |
| `DIGEST_SERVICE_HOST` | Хост Digest Service | `digest-service` |
| `DIGEST_SERVICE_PORT` | Порт Digest Service | `50052` |
| `LOG_LEVEL` | Уровень логирования | `INFO` |

**Примечание**: `TELEGRAM_API_ID` и `TELEGRAM_API_HASH` требуются только для функции импорта папок каналов. Без них бот будет работать, но импорт папок будет недоступен.

## Получение токена бота

1. Найдите [@BotFather](https://t.me/botfather) в Telegram
2. Отправьте команду `/newbot`
3. Следуйте инструкциям для создания бота
4. Скопируйте полученный токен в `.env` файл

## Взаимодействие с другими сервисами

### User Service (gRPC)

Bot Service использует gRPC для взаимодействия с User Service:

- `CreateUser` - создание нового пользователя
- `GetUserProfile` - получение профиля пользователя
- `UpdateUserPreferences` - обновление предпочтений
- `AddSubscription` - добавление подписки на источник
- `RemoveSubscription` - удаление подписки
- `GetUserSubscriptions` - получение списка подписок

### Digest Service (будущее)

Планируется интеграция для:
- Запроса генерации дайджеста по требованию
- Получения истории дайджестов
- Отправки дайджестов пользователям

## FSM (Finite State Machine)

Бот использует FSM для управления состояниями диалога:

### OnboardingStates
- `waiting_for_interests` - ожидание описания интересов
- `waiting_for_sources` - ожидание добавления источников
- `waiting_for_schedule` - ожидание выбора расписания

### SettingsStates
- `waiting_for_new_interests` - ожидание новых интересов
- `waiting_for_new_source` - ожидание нового источника

## Логирование

Логи выводятся в stdout с уровнем, заданным в `LOG_LEVEL`.

Формат лога:
```
%(asctime)s - %(name)s - %(levelname)s - %(message)s
```

Уровни логирования:
- `DEBUG` - детальная отладочная информация
- `INFO` - общая информация о работе
- `WARNING` - предупреждения
- `ERROR` - ошибки

## Обработка ошибок

- Все ошибки gRPC логируются
- Пользователю показываются понятные сообщения об ошибках
- При критических ошибках бот продолжает работу

## Разработка

### Добавление новых команд

1. Создайте новый роутер в `handlers/`
2. Определите обработчики команд
3. Зарегистрируйте роутер в `main.py`

Пример:
```python
from aiogram import Router
from aiogram.filters import Command
from aiogram.types import Message

router = Router()

@router.message(Command("mycommand"))
async def cmd_mycommand(message: Message):
    await message.answer("Hello!")
```

### Добавление новых состояний FSM

```python
from aiogram.fsm.state import State, StatesGroup

class MyStates(StatesGroup):
    waiting_for_input = State()
    processing = State()
```
